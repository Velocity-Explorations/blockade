Every HTTP request costs sats. Watch the protocol in action.

Step 1

Make a request

An unauthenticated request to a gated endpoint returns 402 Payment Required with a payment challenge.

GET http://localhost:8080/get
402 Payment Required

The proxy minted a payment challenge tied to this specific request. No token, no access.

Step 2

Pay the Lightning invoice

Pay the BOLT11 invoice. The preimage released by your node is cryptographic proof of payment.

Step 2

Send on-chain Bitcoin

Send exactly the required amount to the address. The proxy checks the mempool — no block confirmation needed (0-conf).

Prerequisites: run these once before using this mode:
make up-onchain && make setup-onchain
setup-onchain creates the tester wallet with funded regtest coins used by the payment command below.

After running the command, click Check Payment. The proxy verifies the mempool and grants access — no preimage needed for on-chain payments.
