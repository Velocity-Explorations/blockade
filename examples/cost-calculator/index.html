Model: per-request price p(N) = floor + slope·(N−1) for the Nth request under one credential, so cumulative cost grows with the square of volume (superlinear, not exponential). To escape the curve an adversary resets to a fresh staked identity; the cost-minimizing interval is m* = √(2·stake / slope), which holds the amortized cost at √(2·stake·slope) sats per request no matter how they reset. Undefended cost is floor·N. Linear view auto-zooms to where the curve is legible; full-campaign totals are in the panel. Figures are illustrative; the demo runs on signet where sats carry no value.
